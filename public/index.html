Introduction

Welcome to the RPG Hero API documentation. This API allows you to manage a database of RPG heroes and their quests using standard RESTful principles.

The API handles standard CRUD operations, complex validation, and relationships between data entities.

Base URL

All API requests should be made to the following base URL:

http://localhost:3000/

Heroes

Heroes are the main characters in the RPG world. They have attributes like health, mana, strength, and intelligence.

List all Heroes

Returns a paginated list of heroes.

GET /heroes

Parameters

Name Description
limit
integer		 The maximum number of items to return (default: 10).
offset
integer		 The number of items to skip before starting to collect the result set.
sort
string		 The field to sort by (e.g. level, name).
Example Request
curl -G https://localhost:3000/heroes \
    -d limit=5 \
    -d sort=level

Create a Hero

Create a new hero with the provided attributes.

POST /heroes

Properties

Property Description
name *
string		 The name of the hero. Must be unique and contain no numbers.
class *
string		 The hero's class. One of: warrior, mage, rogue, healer.
level
integer		 Level of the hero (1-100). Default is 1.
Example Request
curl -X POST https://localhost:3000/heroes \
    -H "Content-Type: application/json" \
    -d '{
        "name": "Gandalf",
        "class": "mage",
        "level": 100,
        "health": 500,
        "mana": 1000,
        "strength": 20,
        "intelligence": 95
    }'

Quests

Quests are missions available for heroes to undertake.

List all Quests

Returns a paginated list of available quests.

GET /quests
Example Request
curl https://localhost:3000/quests

Create a Quest

POST /quests

Properties

Property Description
title *
string		 Title of the quest.
difficulty *
string		 One of: easy, medium, hard, legendary.

Pagination

All list endpoints support pagination via limit and offset parameters.

This allows you to retrieve large datasets in smaller chunks.

GET /heroes?limit=10&offset=20

Search & Filtering

You can search for resources using specific search endpoints.

GET /heroes/search
Parameter Description
name Partial match on hero name.
class Exact match on hero class.
minLevel Filter heroes above a certain level.
